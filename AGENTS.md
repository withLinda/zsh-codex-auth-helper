# AGENTS.md

Please use simple English to make it easier for non-native speakers to understand, unless you need a specialized term for programming, science, psychology, philosophy, medicine, or another technical field.

When explaining something, prefer clear logic and carefully chosen analogies. The analogy must match the real situation.

## Onboarding Rule

- Before non-trivial work, read `Learnings.md`.
- After any problem or bug that requires extra effort to solve, update `Learnings.md` with the useful lesson.
- Add the lesson while the context is fresh, before the final response when possible.
- Include the symptom, root cause, guardrail, and any command or file path that would help the next agent.
- Do not write secrets, private tokens, personal auth-file contents, or raw sensitive request data into `Learnings.md`.
